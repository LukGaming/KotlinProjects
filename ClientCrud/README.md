# Desafio 30 Dias de Android Nativo

Este repositório faz parte do meu desafio pessoal de **30 dias aprendendo Android Nativo** com Kotlin.

A proposta é simples: estudar e praticar um pouco todos os dias, construindo funcionalidades reais para evoluir de forma consistente.

## Objetivo do desafio

Em 30 dias, quero consolidar os fundamentos de desenvolvimento Android nativo e ganhar segurança para criar aplicações completas, com boas práticas de código e arquitetura.

## Regras do desafio

- Estudar e programar todos os dias
- Registrar o que foi aprendido
- Evoluir o projeto de forma incremental
- Priorizar entendimento real em vez de apenas copiar código

## O que estou praticando

- Kotlin para Android
- UI com XML e componentes nativos
- Navegação entre telas
- Consumo de dados e estado de interface
- Organização em camadas (UI, domínio e dados)
- Boas práticas de código limpo

## Estrutura de progresso

Vou usar este projeto para documentar minha evolução diária:

- Dia 1 ao 10: fundamentos e base da aplicação
- Dia 11 ao 20: funcionalidades intermediárias e refatoração
- Dia 21 ao 30: melhorias, testes e polimento final

## Como acompanhar

Neste README e nos commits, deixarei claro:

- O que foi implementado no dia
- Qual foi o principal aprendizado
- Quais dificuldades apareceram
- Próximos passos

## Tecnologias usadas

- Kotlin
- Android SDK
- Gradle (Kotlin DSL)

## Resultado esperado ao final

Ao final dos 30 dias, quero ter:

- Um projeto Android funcional e evoluído dia após dia
- Mais domínio de Android nativo com Kotlin
- Clareza sobre arquitetura, organização e manutenção de código
